# 영어 회화 마스터 — 제작 명세

## 목적
회화(말하기) 중심 영어 학습 웹앱. 자료: 회화 패턴 500개 + 숙어 400개(원본 #25 누락으로 실제 399개).
플래시카드 + 간격반복(SRS, SM-2 간소화). 항목당 예문 3개. 한→영(말하기) 방향 기본.

## 파일 구조
- `index.html` — 앱 본체 (단일 파일, CSS/JS 포함). `var DB=[]` 선언 후 data/*.js 로드.
- `data/p01.js` ~ `data/p10.js` — 패턴 500개 (파일당 50개, p01=001~050 … p10=451~500)
- `data/i01.js` ~ `data/i08.js` — 숙어 399개 (파일당 50개, i08은 49개)
- 원본(UTF-8 변환본): `패턴_utf8.txt`(1줄=1항목+빈줄), `숙어_utf8.txt`(항목당 4줄)

## 데이터 스키마 (각 data 파일)
```js
DB.push(
{n:"p001",t:"p",e:"I'm afraid of ~",k:"~이 두려워요",x:[["EN","KO"],["EN","KO"],["EN","KO"]]},
{n:"i001",t:"i",e:"abide by",k:"~을 지키다",s:"keep, be faithful to",x:[["EN","KO"],["EN","KO"],["EN","KO"]]}
);
```
- `n`: id (p001~p500 / i001~i399, 원본 순서대로), `t`: "p"=패턴, "i"=숙어
- `e`: 영어 표현 (패턴의 물결표는 ` ~` 형태 유지, S.O→someone, S.T→something 정규화)
- `k`: 한국어 뜻 (원본 기준, 어색한 표기는 자연스럽게 다듬기)
- `s`: 동의어 (숙어만, 원본의 = 부분. 원본 오류는 수정: 예 #24 at first hand =directly)
- `x`: 예문 3개 [영어, 한국어해석]. 문자열은 큰따옴표, 내부 인용은 작은따옴표 사용. 백슬래시 금지.

## 예문 작성 규칙
- 숙어: 원본 예문(오타 수정) 1개 + 새 회화체 예문 2개. 한국어 해석 모두 새로 작성(해요체).
- 패턴: 새 예문 3개 (원본에 예문 없음). 반드시 해당 패턴 사용.
- 길이 5~14단어, 일상 회화체. 상황 다양화(직장/친구/쇼핑/여행/가족/건강/전화).
- 한국어 해석은 자연스러운 해요체.

## 검증
각 데이터 파일 작성 후 `node --check data/pXX.js` 통과 확인. 전체 완료 후 DB.length=899 확인.

## 모바일 (주 사용 환경)
- 모바일 UI: 평가 버튼 화면 하단 고정(sticky), 터치 타깃 확대, iOS 입력 자동확대 방지(16px), safe-area 대응.
- `영어회화마스터.html` = 단일 파일 버전(데이터 18개 병합, 346KB). **데이터 수정 후 `node build_single.js`로 재생성.**
- PWA: manifest.json + icon.svg + sw.js. https로 호스팅하면 홈 화면 설치 + 오프라인 캐시 동작 (file://에서는 자동 비활성).
